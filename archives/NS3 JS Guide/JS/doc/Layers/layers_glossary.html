LAYERS AND JAVASCRIPT EXTENSIONS FOR LAYERS GLOSSARY

This glossary describes elements of HTML and JavaScript syntax needed to implement a Web page using the new layers feature in the Navigator 4.0 component of Netscape Communicator Preview Release. In addition, several simple ideas for ways to enhance Web pages are presented.

The functionality of the new <LAYER> tag allows you to define precisely positioned, overlapping layers of transparent or opaque content in a Web page. With a little JavaScript code, you can move layers, hide them, and change the order in which they overlap. With layers you can use HTML and JavaScript to create dynamic animations on a Web page.

The Navigator 4.0 component in the Preview Release extends the objects and methods available in JavaScript to contain a Layer object and a Layers array that enumerates all the layers in a document. This allows dynamic control of the position, visibility, and z-order (stacking order) of layers.

As future versions of Netscape Communicator are released, additonal tags and functionalities will be added to this glossary. Check back here for updates. For more information, visit DevEdge Online.

LAYER
The <LAYER> tag starts a layer, and the </LAYER> tag ends the layer. All the content between the opening and closing layer tag can be treated as a single layer of content that can be moved, clipped, and altered in various ways. For example, you could have many overlapping layers that can be dynamically peeled away to reveal the layers underneath.

Layers have a stacking order that determine which layer appears on top of another for overlapping layers. Using JavaScript you can dynamically change whether a layer is visible or not, which means you can hide a layer and make it reappear again if you want.

Each pixel in a layer can be transparent or solid. If a pixel is transparent, the content of underlying layers shows through it. You can specify background images and background colors for layers just as you do for body page. Specifying a background color causes the entire layer to become opaque. Tiled background images can contain transparent pixels.

If you resize a browser while it is showing a page containing layers that have been dynamically changed by JavaScript, the layers reset to their initial state. That is, they reset to the state they were in when the page was first loaded.

NAME="layerName"
This is an identification tag for the layer. Layer names must begin with an alphabetic character. The name is used for referring to the layer from within HTML and external scripting languages such as JavaScript. Each layer name within a document must be unique. This attribute is optional; by default all layers are unnamed.

LEFT=xPosition

TOP=yPosition
The LEFT and TOP attributes specify the pixel position or relative position (in percent) of the layer in terms of the horizontal and vertical position of the top left corner of the layer. The origin is the upper left corner of the parent layer or document, with coordinates increasing downwards and to the right. The values must be integer values in the range [-2147483648, 2147483647]; these are the limits of a 32-bit, signed integer. The default values for both LEFT and TOP correspond to the position of the content of the tag if the tag did not exist. That is, the current layout position when the tag is encountered.
   <LAYER LEFT=20 TOP=100>

   This layer appears 20 pixels in from the left and 100 pixels down from the top of the parent layer.

   </LAYER>

Both attributes are optional. After the <LAYER> tag is encountered, subsequent HTML layout will resume at the XY location of layout before the <LAYER> tag was encountered.

WIDTH=layerWidth
The WIDTH attribute controls how the contents of the <LAYER> tag will be wrapped. It is specified in terms of pixels and must be an integer in the range [-2147483648, 2147483647]; these are the limits of a 32-bit, signed integer, or as a percentage. If elements that cannot be wrapped, such as images, extend beyond the width specified, the actual width of the contents of the layer will be larger than the specified value. This behavior is analogous to the way in which the width of a window sets the default wrapping for any HTML it contains. The attribute is optional; in the absence of a WIDTH specification, the layer contents wrap at the right boundary of the enclosing block.

HEIGHT=layerHeight
HEIGHT acts as a reference length for attribute values in child layers that are specified using percentages.

CLIP="x1_offset, y1_offset, x2_offset, y2_offset" or CLIP="x2_offset, y2_offset"
The CLIP attribute specifies a rectangle containing the portion of the contents to be drawn. Everything outside this rectangle is drawn as transparent and so is invisible. This clip is expressed in a coordinate system where the LEFT and TOP of the layer space is the origin.

The CLIP attribute is specified with four numbers, in pixels, and must be an integer in the range [-2147483648, 2147483647]; these are the limits of a 32-bit, signed integer, or as a percentage. These are the x1_offset, y1_offset, x2_offset, and y2_offset, where the x1 and y1 offsets specify the upper left corner of the displayed region relative to the layer's LEFT and TOP position values, and the x2 and y2 offsets specify the lower right corner. So, the numbers specified in the attribute from left to right should be the horizontal left value, the vertical top value, the horizontal right value, and the vertical bottom value.

If x1_offset and y1_offset are omitted, they default to (0,0) (that is, the coordinates specified by the layer's LEFT and TOP attributes). In this case, the x2 and y2 offsets still represent the lower right corner. If CLIP is omitted entirely, x2_offset and y2_offset are chosen such that everything within the layer is unclipped - that is, visible.

Z-INDEX=layerZ

ABOVE="layerName"

BELOW="layerName"
The Z-INDEX, ABOVE, and BELOW attributes specify the z-order (stacking order) of layers, overriding the default behavior of placing new layers on top of all existing layers. Only one of the Z-INDEX, ABOVE, or BELOW attributes may be used for a given layer.
The Z-INDEX attribute allows a layer's z-order to be specified in terms of a positive integer. Layers with higher-numbered layerZ values are stacked above those with lower ones.
The ABOVE attribute specifies the NAME attribute of the layer immediately on top of a newly created layer. In other words the new layer is created just below the layer specified by the ABOVE attribute. Similarly, the BELOW attribute identifies the NAME attribute of the layer that will be immediately underneath the newly created layer. For either attribute, the specified layer must already have been created. Forward references to other layers are not permitted and will result in the default layer-creation behavior (as if the ABOVE or BELOW attribute had not appeared).

VISIBILITY=SHOW | HIDE | INHERIT
The VISIBILITY attribute is used to determine whether the layer is initially displayed. SHOW, which is not implemented in the preview release, indicates that the layer should be drawn, HIDE indicates that the layer should be hidden, and INHERIT indicates that the layer's visibility should always be the same as its parent layer's visibility. The attribute is optional; by default, VISIBILITY=INHERIT. The hidden state of a layer can be changed dynamically by using a scripting language.

BGCOLOR="rgbColor"

BACKGROUND="imageURL"
These attributes control the solid background color or tiled image backdrop of a layer. They are similar to the BGCOLOR and BACKGROUND attributes of the <BODY> tag. BGCOLOR is either the name of a standard color or an RGB value. BACKGROUND is the URL of a backdrop image. Backdrop images may contain transparent pixels. Both of these attributes are optional. By default, a layer is transparent - layers below a layer with neither attribute show through in the areas in which a background color or pattern would have been visible.

SRC=URL
All the content in the page referenced by URL is placed in the layer.

NOLAYER
All HTML between the <NOLAYER> and </NOLAYER> is ignored in Communicator.




LAYERS IN HTML: IDEA GLOSSARY

This section illustrates several simple ways in which Web page layout has been enhanced by the addition of the <LAYER> tag. This section is by no means comprehensive; rather, it contains enough ideas to start working with layers.

Layout Control

The inability to control the positions of HTML elements has become an issue for media-rich content. Before the advent of layers, Navigator laid out HTML objects in a sequential, planar fashion. Now, by specifying the upper left corner of a layer using the LEFT and TOP attributes, HTML blocks can be precisely positioned inside Navigator's window. For example, a layer containing an image and <H1> formatted text can be specified to appear at coordinates 0,20. Subsequent HTML layers can be positioned precisely relative to this layer. Additionally, layers can be made to appear above or below other HTML elements on the page.


Background Images and Colors

In previous versions of Navigator, background images and colors were restricted to the background of the entire document or to tables within a document. With layers, multiple parts of a page can have different background colors and background patterns. Transparency and the overlapping of multiple layers can be used to achieve even more interesting visual effects.


Nested Layers

Nested layers without JavaScript could be used as a layout method for hierarchical documents in the same way that entire HTML tables are often placed within table cells. For example, to make something that looked like an aquarium, you would need a solid-colored rectangle to serve as a fish tank and some fish images. It might look like this: 

     <!-- Define a layer with a background to serve as the "fish tank"-->
     <LAYER NAME="FISHTANK" BACKGROUND="LIGHT BLUE" CLIP=500,200>
       <LAYER TOP=100 LEFT=100>
           <IMG SRC="grouper.gif">
       </LAYER>
       <LAYER TOP=200 LEFT=200>
           <IMG SRC="grouper.gif">
       </LAYER>
       <LAYER TOP=200 LEFT=200>=>
           <IMG SRC="lionfish.gif">
       </LAYER>
     </LAYER>

The positions of all the fish images are defined relative to the parent "FISHTANK" layer, so if you decided to change the placement of the fish tank on the page, you wouldn't have to change the coordinates of each individual fish image - they would all move with the fish tank layer.

Note that hiding a parent hides the children if their VISIBILITY attribute is INHERIT.


Hidden Layers

The hidden property of the <LAYER> tag allows certain layers to be displayed or hidden initially. By adding simple JavaScript statements, one can produce dynamic hiding effects with layers. However, hidden layers can be useful even without JavaScript. Consider, for example, a developer who wishes to hide HTML form elements without using the HIDDEN form input type. By using hidden layers, the developer could hide entire form elements (for example, text and selection entries) from view.

SYNTAX GLOSSARY

LAYER OBJECT

Layer objects, which are created automatically, are associated with sections of an HTML document that are contained within <LAYER> tags. A Layer object has properties that reflect the corresponding HTML layer's properties. Some layer properties can be directly modified by assignment. For example, mylayer.visibility = "hide" causes a visible layer to become hidden. In addition, a Layer object has methods that can affect these properties.

As with all JavaScript objects, a Layer object's properties and methods can be accessed in the following manner:

  1. layerName.propertyName
  2. layerName.methodName(parameters)

layerName refers to a Layer object, propertyName is one of the properties described below, methodName is one of the methods described below, and parameters are the parameters associated with a given method.

LAYER OBJECT PROPERTIES

visibility yes "hide" or "inherit" (these are strings, not reserved words).
clip.left
clip.top
clip.right
clip.bottom
clip.width
clip.height		 yes Defines the clipping rectangle, which specifies the part of a layer that is visible.
siblingAbove no The sibling layer above the one being referenced, in z-order, or null if the layer is topmost.
siblingBelow no The sibling layer below the one being referenced, in z-order, or null if the layer is bottommost.

LAYER OBJECT METHODS

Method Short Description

moveTo(x, y) Translates a layer such that its upper left corner is (x,y).
resizeTo(width, height)

LAYERS ARRAY

A layer can be referenced in JavaScript using the Layers array, which contains an entry for each object that has been created. Objects in the Layers array can be accessed using either a non-negative integer index or by using the name of the layer specified by its NAME attribute. When accessed using an integer index, the array elements appear in z-order from back to front, with zero being the bottommost layer and higher layers indexed by consecutive integers. Note that the array index of a layer is not the same as its z property, because the latter does not necessarily enumerate layers with unique consecutive integers.

The following represent valid ways of accessing Layer objects:
document.layerName
document.layers[index]
document.layers["layerName"]

Elements of the Layers array are JavaScript objects that cannot be set by assignment, though their properties can be set. For example, the statement

document.layers[0]="music"

is invalid and will be ignored because it attempts to alter the Layers array. However, some properties of the objects in the array are both readable and writable. Thus, for example, the statement

document.layers["somelayer"].left = 100

is perfectly valid; it represents the intended use of the Layers array.

The number of layers in a document can be obtained from the length property of the Layers array:

document.layers.length

Note: in PR3 and later, each layer has its own document property, similar to the document property of a window. Therefore, to access an image in a layer, a script outside the layer would use:

window.document.layers["layerName"].document.image[m]

See also the Array object in the JavaScript Authoring Guide.

IDEA GLOSSARY

HIDING AND SHOWING LAYERS

A broad spectrum of interesting and useful visual effects can be achieved by hiding and showing layers using the visibility property. Layers that have been defined in HTML with the attribute VISIBILITY set to HIDE are initially hidden when a page is loaded. By setting the VISIBILITY attribute of a hidden layer to SHOW, you can make layers (containing arbitrary HTML) appear suddenly via the use of JavaScript events. Similarly, visible layers can go away, either temporarily or permanently.

A simple example of this might be a form containing a checkbox labeled "Add me to your mailing list." Initially this checkbox appears all by itself. When the user checks the checkbox, a text entry field for the user's email address appears - it isn't necessary to reload the page.

By using several hidden layers, you can achieve even more effects. For example, you can create a movie with variable frame rates by hiding and showing several layers containing static images in a sequence (use of this animation effect is described below). Another example might be a page containing a paragraph of text that changes when the user's mouse passes over various menu items.

MOVING LAYERS

Using the Layer object's left and right properties, the offset() method, or the moveTo() method, layers can be dynamically moved after a page has been drawn. Visible layers can be made to jump across the page, and hidden layers can be moved into position before they are displayed.

Consider, for example, a puzzle consisting of a three-by-three grid containing eight tiles, each of which is a piece of a larger picture. Because there are nine squares and only eight tiles, there will always be one empty square. When the user clicks on a tile adjacent to the empty square, the tile changes places with the empty square. In this way a "shuffled" image can be repeatedly rearranged until the correctly configured image is constructed. To implement this, you would make each tile an image within a layer. When the user clicks on a tile (we can tell this by using the onClick() event handler), its layer is repositioned so that the tile appears over the empty square.

As another example, consider how balloon help might be implemented. A layer is drawn containing a large, empty "balloon," much like the ones used in cartoons. Additionally, for each area on a page, several layers are defined using chunks of HTML that describe the area. All these layers are initially hidden. When the user moves the mouse over one of the page areas (we can tell this by using the onMouseOver() and onMouseOut() event handlers in conjunction with a layer containing a totally transparent image), the balloon layer is repositioned over that area of the page. Then, the layer with the info for that area of the page is repositioned over the balloon layer. Finally, both the balloon layer and the info layer are made visible.

Please note that in Preview Release 1, these event handlers only apply to imagemaps. In future releases, they could be event handlers for layers.

RECLIPPING LAYERS

The Layer object's clip property and resize() method allow the portion of a layer that is displayed to be changed.

A pull-down menu is an example of this. To create a pull-down menu, a layer is defined that shows a complete menu, including its title. The CLIP attribute of this layer is defined in such a way that only the title of the menu is visible when the page is initially loaded. Subsequently, when the user clicks the title of the menu, the layer's clipping is changed so that the entire menu is visible. When the user then clicks one of the menu items, the menu layer's clip property is reset to its original value.

By combining the reclipping method described in the above example with a layer's left and right properties, you can create "scrollable" layers. As an example, consider a layer containing a very large amount of HTML. To fit into a page's layout, only a small part of this HTML is made visible by clipping the layer. Next to the clipped layer is a down arrow image. When the user clicks the arrow, the layer is reclipped so that the visible portion of the layer is 10 pixels lower within the layer. To compensate for the movement of the visible part of the layer, the entire layer moves up 10 pixels. The net effect of these changes is that the content of the layer seems to scroll whenever the down arrow is clicked.

ANIMATION

Under the right circumstances, repeatedly performing any of the functions described above results in layer animation. For example, layers need not jump across the page when they are moved. If a layer is moved from its original position to its final position in increments, it seems to slide across the page. You can control the sliding speed by adjusting how far the layer moves in each increment and how much time passes between increments.

To animate a layer, you use the setTimeout() method within a recursive function to separate each modification to the layer's properties. Consider the following example code:

function moveSomeLayer()
   code which performs one movement of your layer(s) goes here

function layer_timeout()
   moveSomeLayer();
   set Timeout("layer_timeout()",100);

layer_timout();

Each of the following examples illustrates animation in conjunction with hiding and showing layers, moving layers, and reclipping layers. Imagine that a whole section of HTML needs to fade away gradually until it disappears. This HTML content appears on top of a solid white background. To achieve this effect, nine GIF images are created. Each image has a certain percentage of opaque white pixels; the remaining pixels are transparent. The opaque pixels are distributed within the image in a pattern. The images contain 10, 20, 30, 40, 50, 60, 70, 80, 90, and 100 percent opaque pixels. Each image is used as the background pattern in a hidden layer. Each hidden layer has the same dimensions and position as the section of HTML that needs to be faded. By rapidly hiding and showing these layers in succession until the 100 percent layer is visible, the HTML content appears to fade into white.

For the next example, imagine that a GIF image of a ball needs to be made to bounce around a certain region of a Web page. First, a layer containing the ball image is created. Next, the LEFT and TOP attributes of the layer are stored in global variables. Then, four functions are written to move the layer a small number of pixels diagonally up and to the right, down and to the right, down and to the left, and up and to the left using the offset() method. Finally, these functions are called in succession to create the desired bouncing ball action.

Finally, consider the pull-down menu example described in the Reclipping Layers section above. Rather than having the menu pop up instantly, the menu can be made to scroll open. This effect can be achieved by repeatedly resetting the layer's clip.bottom property.
